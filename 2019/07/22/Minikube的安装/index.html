运行minikube程序创建k8s

通过 minikube starat 去创建k8s环境。

通过kubectl cluster-info 去连一下k8s api server.

此时并不代表，整个k8s集群搭建好了，因为k8s里的服务还需要起，比如API server，scheduler，kubelet等等，它们都是以容器的方式在后台启动。可以通过minikube ssh进到虚机里，然后看看是否有一些container运行起来了
